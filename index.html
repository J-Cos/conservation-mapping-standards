How Accurate Is Your Map?

Maps without rigorous accuracy assessment risk misleading real-world decisions. This interactive demonstrator accompanies the spatial mapping standard proposed by Schulte to Bühne et al. (2026). Walk through the best-practice pipeline on synthetic data — then see what happens when common shortcuts are taken.

1

Synthetic Data Generation

✓ Generate synthetic landscape
Pending

First, we generate satellite imagery for an artificial landscape (1000 by 1000 pixels). The satellite imagery has 10 bands (similar to many open-source satellite sensors), and realistic spatial relationships of reflectance values. We also generate the variable that we want to map — either land cover or above-ground biomass — by combining the artificial satellite bands and adding noise (simulating real sensor measurement error). This means that we know what the true values are at every point on this map, and can determine how well any map derived from the satellite imagery reflects them.

Technical details
Spectral bands
10 bands mimicking a multispectral satellite sensor: Blue, Green, Red, NIR, SWIR-1, SWIR-2, Thermal, plus derived indices (NDVI, NDWI) and a texture layer. Units are dimensionless reflectance values (0–1).
Spatial autocorrelation
Generated using Perlin noise with fractal Brownian motion (fBm) — a standard technique for producing smooth, spatially correlated random fields at multiple scales.
False-colour composite
The left image maps NIR → Red, Red → Green, Green → Blue. Vegetation appears red/orange because it reflects strongly in the near-infrared.
Categorical mode: hidden environmental variable
Class boundaries are influenced by an unobservable environmental gradient (e.g. soil moisture) that the classifier cannot access. This mimics the real-world gap between what a sensor measures and what determines land cover.
Continuous mode: biomass generation
Biomass (0–500 Mg/ha) is derived directly from nonlinear combinations of the spectral bands with added heteroscedastic noise. Unlike categorical mode, no hidden environmental variable is used — the mapping difficulty arises from sensor noise and the nonlinear spectral–biomass relationship.
Sensor noise
Additive random noise is applied to all bands to simulate real sensor measurement error. The classifier trains on these noisy bands, while the true values are derived from clean bands — creating a realistic accuracy ceiling.

Spatial Blocking & Sampling Design

✓ Ensure spatial independence
Pending

Divide the study area into spatial blocks to ensure training and validation data are independent. Nearby pixels are similar (spatially autocorrelatedSpatial autocorrelation: nearby locations tend to have similar values. In remote sensing, adjacent pixels share lighting, topography, and land cover — so they are not independent samples.), so if they end up in both the training and test set, accuracy will be artificially inflated. By grouping pixels into blocks and splitting at the block level, we prevent this data leakageData leakage occurs when information from the test set "leaks" into training, making the model appear more accurate than it truly is on new, unseen data.. Gold-highlighted blocks below are held out for validation in one example iteration (~36.8% of total — this is the mathematical expectation from bootstrap sampling, where each block has a probability of (1 − 1/N)N ≈ 1/e ≈ 36.8% of never being selected).

Technical details
Spatial blocking
Also called spatial cross-validation. Pixels are grouped into square blocks; entire blocks are assigned to either the training or validation set — never split. This ensures that nearby pixels cannot appear in both sets.
Out-of-bag (OOB) blocks
In each bootstrap replicate, ~36.8% of blocks are not drawn ("out of bag"). These unseen blocks form the validation set. The model is never tested on data it trained on.
Block size
Should be larger than the spatial autocorrelation range. Larger blocks = more conservative (less leakage risk) but fewer independent validation samples.
Why not random pixel splitting?
Random splitting ignores spatial structure. Adjacent pixels end up in both train and test sets, letting the model "cheat" by memorising local patterns rather than learning generalisable relationships.

Model Training with Repeated Cross-Validation

✓ Quantify map accuracy & precision
Pending

A single train/test split can give a misleading accuracy estimate. Instead, draw B bootstrap samplesBootstrap: a statistical resampling technique. Draw N items from your dataset with replacement to create a new sample of the same size, then repeat many times to estimate variability. of spatial blocks (with replacement). For each replicate, train a Random ForestRandom Forest: a machine learning algorithm that builds many decision trees on random subsets of the data, then combines their predictions by majority vote (classification) or averaging (regression). on the selected blocks and validate on the ~36.8% of blocks held out. This produces B independent accuracy estimates from which distributions and confidence intervalsA 95% confidence interval means: if we repeated this entire process many times, 95% of the intervals would contain the true value. Wider intervals = less certainty. are derived — revealing how certain we can be about map quality.

Technical details
Bootstrap resampling
A bootstrap sample is created by choosing, at random, blocks with replacement from the total pool of N blocks. On average, ~63.2% of blocks are drawn at least once (training set); the remaining ~36.8% are out-of-bag (validation set).
Random Forest
An ensemble of T decision trees. Each tree is trained on a bootstrap sample of the training data and a random subset of features (mtry = √p for classification, p/3 for regression). Predictions are combined by majority vote or averaging.
Repeat assessments (B)
More replicates produce narrower confidence intervals and more precise accuracy estimates. 100+ replicates are recommended; 20–50 suffice for quick exploration.

Accuracy Assessment

✓ Assess accuracy with confidence intervals
Pending

Distributions of accuracy metrics across all B replicates. 95% confidence intervals indicate the precision with which accuracy is known. Class-wise user's accuracyUser's Accuracy (UA): Of all pixels the map labels as class X, what proportion are actually class X? Also called precision or reliability. Answers: "If the map says it's forest, can I trust that?" and producer's accuracyProducer's Accuracy (PA): Of all pixels that truly are class X, what proportion did the map correctly identify? Also called recall or sensitivity. Answers: "How much of the real forest did the map find?" ≥ 85% is considered good; ≥ 95% is excellent. Threshold values: (R²Coefficient of determination: the proportion of variance in the true values explained by the model's predictions. R² = 1.0 is perfect; R² > 0.8 is generally considered good for remote sensing. > 0.8, relative RMSERelative Root Mean Square Error: the RMSE divided by the mean of the true values, expressed as a percentage. Lower is better; < 20% is generally acceptable. < 20%). Confidence intervals should not cross these thresholds.

A large confidence interval around a high accuracy metric means that we cannot say confidently that the accuracy metric truly is high. The smaller the confidence interval, the more certain we can be that a map created by a single model drawn from this sample gives a reliable representation of the truth.

Technical details
Overall accuracy
The proportion of all validation pixels correctly classified. A single number summary, but can be misleading if classes are imbalanced.
User's accuracy (precision)
Per-class metric: of all pixels predicted as class X, what fraction truly are X? Low UA = the map is labelling many non-X pixels as X (commission error / false positives).
Producer's accuracy (recall)
Per-class metric: of all pixels that truly are class X, what fraction did the map correctly identify? Low PA = the map is missing many X pixels (omission error / false negatives).
Error matrix (confusion matrix)
A table showing predicted vs. true class for every validation pixel. Each row is a true class; each column is a predicted class. Diagonal cells are correct; off-diagonal are errors. The basis for computing UA, PA, and overall accuracy.
R² (coefficient of determination)
Proportion of variance in the true values explained by the model. R² = 1 − (SS_res / SS_tot).
RMSE (Root Mean Square Error)
The square root of the average squared difference between predicted and true values. Same units as the response variable (Mg/ha for biomass).

Summary Statistics & Uncertainty

✓ Error-corrected summary statistics
Pending

Why does this matter? Simply counting pixels per class ignores classification errors and gives biased area estimates. Instead, the error matrix from each assessment replicate is used to correct class areas (Olofsson et al. 2014Olofsson et al. (2014) "Good practices for estimating area and assessing accuracy of land change" — the standard reference for error-corrected area estimation using stratified estimation with the error matrix.), producing error-corrected area estimates with 95% confidence intervals. Compare the corrected vs naive estimates below. Each replicate predicts biomass for the entire map. The distribution of predicted total biomass across replicates quantifies uncertainty, and is compared against the known true total.

Large confidence intervals mean that we have very little idea of the true value, with intervals sometimes spanning orders of magnitude. Over- or underestimating map-derived variables, such as the extent of forest or total biomass, to this extent is likely to have significant implications for conservation decisions or downstream analyses that use this data.

Technical details
Why not just count pixels?
If a map is 90% accurate and covers 1,000 km², naively counting pixels labelled "forest" ignores the 10% of pixels that are wrong. Some non-forest pixels are incorrectly labelled forest (inflating the estimate), while some real forest pixels are missed (deflating it). These errors rarely cancel out.
Error-corrected area estimation
Uses the error matrix to adjust raw pixel counts. The matrix tells us: of pixels labelled "forest", what fraction truly are forest? This correction removes systematic bias from the area estimate.
Confidence intervals
Each bootstrap replicate produces a different corrected area estimate. The distribution of these estimates gives a 95% CI showing the range within which the true area likely falls.
Naive area (pixel counting)
Shown in red for comparison. This is what you get if you simply count how many pixels the map assigns to each class — ignoring errors entirely.
